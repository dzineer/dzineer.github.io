The grammar is the guardrail

Safety by expression, not by supervision.

Most AI safety tries to make models not want to do bad things. Gibber asks a different question: how do we build a system where bad things aren't expressible? What's not in the grammar cannot exist — and what cannot exist cannot be chosen.

1Grammar

What's not in the AST cannot exist. Malformed forms are rejected at parse time before anything runs.

2Dictionary

Every symbol has one fixed meaning. No synonym exploitation, no prompt-injection through ambiguity.

3Typed tool contract

Inputs constrained by enums. Outputs are a closed §one-of. Weirdness never reaches the agent.

4Exhaustive match

Every execution path is named. No unnamed fallback, no gap for creative reinterpretation.

5Auditability

Every action is a walkable AST node. Trace exactly what happened, in what order, with what I/O.

“You’ve essentially described a capability containment system rather than a value alignment system. And containment is a solved engineering problem. Alignment isn’t.”

— Claude, on Gibber’s safety model

“It’s the difference between telling someone dangerous not to pick up a weapon, and building a room where weapons cannot physically exist.”

— Claude, on grammar as containment

Eidetic Memory

Persistent memory engine for AI agents

Dual-mode memory system: long-term wiki-graph plus short-term cache. Deep code understanding via Tree-sitter. Hybrid recall pipeline with RRF and deep traversal. Built in Rust for speed. Surfaces as CLI, MCP server, and web UI.

How Eidetic remembers

A living architecture for agent memory.

An AI agent's query flows through a tiered memory system: a short-term cache for recent context, a persistent wiki graph for long-term knowledge, and a semantic vector index for similarity. Their results merge in a hybrid recall layer and return as ranked wiki documents.

STM cache (short-term, fast, volatile)Wiki graph (long-term, persistent)Vector index (HNSW, semantic)Hybrid recall (RRF + deep traversal)
How agents find each other

The A2A communication stack.

Agent A wants to talk to Agent B. It uses the A2A Agent Service library to query the Registry for Agent B's address, then sends a secure mTLS JSON-RPC message to the Multi-Agent Host, which routes it to the right agent instance. The full loop plays on repeat below.
